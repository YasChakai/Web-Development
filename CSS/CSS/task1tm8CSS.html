Q1. Je kunt op 3 verschillende manieren CSS in HTML inladen. Welke zijn dat en leg ze uit:

External CSS: je kunt het uiterlijk van een hele website veranderen door slechts één bestand te wijzigen.

Elke HTML-pagina moet een verwijzing bevatten naar het externe stijlbladbestand in het <link> -element, in de head-sectie. Een externe stylesheet kan in elke teksteditor worden geschreven en moet worden opgeslagen met de extensie .css.

Internal CSS: Een interne stijlpagina kan worden gebruikt als een enkele HTML-pagina een unieke stijl heeft. De interne stijl wordt gedefinieerd in het <style> -element, in de head-sectie.

Inline CSS: Een inline stijl kan worden gebruikt om een unieke stijl toe te passen op een enkel element. Om inline stijlen te gebruiken, voegt u het style-attribuut toe aan het relevante element. Het style-attribuut kan elke CSS-eigenschap bevatten.

Q8. Leg in eigen worden uit wat Cascade en Inhertance betekent met voorbeelden.

Stylesheets cascade - op een heel eenvoudig niveau betekent dit dat de volgorde van CSS-regels ertoe doet; wanneer twee regels van toepassing zijn die dezelfde specificiteit hebben, wordt degene die als laatste in de CSS komt, degene die zal worden gebruikt. In het onderstaande voorbeeld hebben we twee regels die van toepassing kunnen zijn op de h1. De h1 wordt uiteindelijk blauw gekleurd - deze regels hebben een identieke selector en hebben daarom dezelfde specificiteit, dus de laatste in de bronvolgorde wint.

Dit is mijn heading.

Specificity is hoe de browser beslist welke regel van toepassing is als meerdere regels verschillende selectors hebben, maar kan nog steeds van toepassing zijn op hetzelfde element.
Het is in feite een maatstaf voor hoe specifiek de selectie van een selector zal zijn: Een elementenselector is minder specifiek het selecteert alle elementen van dat type die op een pagina verschijnen En krijgt dus een lagere score.
Een klassenkiezer is specifieker - hij selecteert alleen de elementen op een pagina die een specifieke klasse-attribuutwaarde hebben En krijgt dus een hogere score.
Voorbeeld tijd! Hieronder hebben we weer twee regels die van toepassing kunnen zijn op de h1. De onderstaande h1 wordt uiteindelijk rood gekleurd - de klasseselector geeft zijn regel een hogere specificiteit, en dus zal deze worden toegepast ook al verschijnt de regel met de elementenselector verder naar beneden in de bronvolgorde.

Inheritance moet ook in deze context worden begrepen sommige CSS-Property en die op bovenliggende elementen zijn ingesteld, worden geërfd door hun onderliggende elementen en andere niet.
Als u bijvoorbeeld een kleur en lettertypefamilie instelt op een element, wordt elk element erin ook gestileerd met die kleur en lettertype,
Tenzij u er rechtstreeks andere kleur- en lettertypewaarden op hebt toegepast.
Omdat het lichaam een ​​blauwe kleur heeft gekregen, wordt dit via de nakomelingen overgeërfd.
We kunnen de kleur veranderen door het element te targeten met een selector, zoals deze span.
